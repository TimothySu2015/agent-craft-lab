# Anthropic: Claude 4.x 系列

Claude 模型以其強大的長上下文處理能力、細緻的指令遵循和對 XML 標籤的偏好而聞名。

## 核心特點

- **偏好 XML 標籤**：Claude 在訓練時接觸了大量 XML，因此使用 `<tag></tag>` 來結構化提示，效果特別好。
- **精確且保守**：Claude 傾向於嚴格遵循字面指令，不會像其他模型那樣經常「過度解讀」或提供未被要求的額外資訊。你需要明確地要求它執行或擴展思考。
- **強大的長程推理**：尤其 Claude Opus，非常擅長在極長的文件中追蹤細節和保持上下文連貫性。

## 官方建議的實用技巧

1. **清晰直接**：這是最重要的。
2. **使用範例**：提供高品質的範例。
3. **XML 標籤**：用標籤包裝指令、上下文、範例等。
4. **鏈式思考**：要求它在回答前先 `thinking`。
5. **預填回應**：以你想要的格式開頭，例如寫下 `{"` 來觸發 JSON 輸出。

## 程式碼生成與探索指引

以下是 Anthropic 提供的高品質 prompt，用於指導 Claude 進行程式碼相關任務，避免幻覺或過度工程化。

**避免幻覺：**
```xml
<investigate_before_answering>
在提出程式碼編輯建議之前，務必先閱讀並理解相關檔案。
不要猜測你沒有檢查過的程式碼。
如果使用者提到特定的檔案或路徑，你必須先打開它再進行解釋。
</investigate_before_answering>
```

**避免過度工程：**
```xml
<avoid_overengineering>
避免過度工程。只做被直接要求或明確必要的改動。
- 不要添加未被要求的功能或進行「改進」。
- 不要為不會發生的場景添加錯誤處理。
- 不要為一次性操作創建輔助函式或抽象。
- 為當前任務使用最小的複雜度。
</avoid_overengineering>
```

## Claude 4.x 專屬最佳實踐

Claude 4.x 系列（包括 Sonnet 4.5、Haiku 4.5、Opus 4.5）相較於前代模型的特點：

| 特點 | 說明 |
|------|------|
| **更精確的指令遵循** | 對明確指令的遵循更加精確，較少「自作主張」添加額外內容。 |
| **需要明確請求額外行為** | 如果你希望獲得「超越預期」的回答，需要在提示中明確請求。 |
| **卓越的長期推理** | Claude 4.5 在長期任務中有出色的狀態追蹤能力，能保持漸進式的進展。 |
| **平行工具執行** | Sonnet 4.5 特別擅長同時執行多個工具操作，提高效率。 |

## Claude 4.x 提示詞建議

- 如果需要詳細解釋，明確要求：「請提供詳細的解釋和背景資訊」
- 如果需要創意發揮，明確要求：「請提供多種可能的解決方案」
- 對於複雜任務，可以請求分步驟思考：「請先分析問題，再提出解決方案」
