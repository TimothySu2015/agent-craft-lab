# OpenAI: GPT 系列 (GPT-5, GPT-4o)

GPT-5 於 2025 年 8 月發布，是 OpenAI 迄今最強大的模型，在數學、程式碼和多模態理解方面大幅超越前代。

## GPT-5 核心特點

- **統一架構**：GPT-5 是一個統一系統，能自動判斷何時快速回應、何時需要深度思考。
- **雙模式切換**：
  - **Instant Mode**：快速回應，適合簡單問答
  - **Thinking Mode**：深度推理，適合複雜任務（類似 o1）
- **卓越性能**：
  - AIME 2025 數學測試：94.6%
  - SWE-bench 程式碼：74.9%
  - MMMU 多模態理解：84.2%

## GPT-5 vs GPT-4o 選擇指南

| 場景 | 推薦模型 | 原因 |
|------|----------|------|
| 複雜推理、數學、程式碼 | GPT-5 (Thinking) | 深度推理能力更強 |
| 快速問答、簡單任務 | GPT-5 (Instant) 或 GPT-4o | 低延遲、成本效益 |
| 成本敏感的批次處理 | GPT-4o | 價格較低 |

## GPT-5.1 / 5.2 更新

- **GPT-5.1（2025年11月）**：改進回應速度和風格、明確 Instant/Thinking 模式選擇、更精確的指令遵循
- **GPT-5.2（2025年12月）**：進一步提升多模態理解能力、改進長上下文處理

## 通用最佳實踐

- **指令遵循能力強**：對於清晰、直接的指令，GPT 系列的遵循能力非常出色。通常不需要用複雜的格式或技巧來引導。
- **專用 `tools` 欄位**：在 API 中，使用專門的 `tools` 和 `tool_choice` 參數來定義和控制外部函式，效果和可靠性遠超在文字提示中手動描述工具。
- **System Prompt 的重要性**：System Prompt 對於設定模型的行為、角色和限制至關重要。

## 工具描述最佳實踐

| 元素 | 好的寫法 | 差的寫法 |
|---|---|---|
| **做什麼** | `查詢用戶的訂單歷史紀錄。` | `透過 API 呼叫資料庫。` |
| **何時用** | `當用戶詢問特定地點的即時天氣時使用。` | `一個天氣工具。` |
| **參數說明** | `location: 城市和州名，例如 "San Francisco, CA"。` | `location: 地點。` |
